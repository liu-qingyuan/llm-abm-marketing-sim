# PRDs

本目录保存 Product Requirements Documents（PRD）。PRD 描述要解决的问题、用户故事、实现决策、测试决策、非目标和后续 issue plan。

## 使用规则

- PRD 应发布到 GitHub issue tracker，并在本地保留 Markdown 副本。
- PRD 不直接承载实现代码，也不替代 GitHub issues。
- 可执行工作应从 PRD 拆成 `ready-for-agent` issues。
